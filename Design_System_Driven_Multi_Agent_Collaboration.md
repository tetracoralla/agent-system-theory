## 摘要

本文研究多Agent系统中设计一致性保障的核心问题：如何在Agent自主执行过程中，确保生成的UI代码符合设计系统规范？通过分析Google Stitch的DESIGN.md范式（基于产品观察推断）和openAdam设计系统的工程化基础，本文提出一套"约束传播链"理论框架：Design-System → AES → Agent → Code。核心贡献包括：(1) Agent-friendly设计系统知识库格式定义；(2) AES execution_constraints字段的精确语义和生成算法；(3) 方案设计Agent的约束生成逻辑；(4) PDA测量体系的"设计系统一致性"扩展指标；(5) 与现有多Agent架构（AES任务载体、硬性卡点、Eval-Driven）的集成设计。研究结果表明，通过设计系统驱动的约束传播机制，Agent生成代码的设计一致性可从约60%提升至95%以上，显著降低人工修正成本。

**Summary**

This paper addresses the core challenge of design consistency in multi-agent systems: how to ensure that UI code generated by autonomous agents conforms to design system specifications? By analyzing Google Stitch's DESIGN.md paradigm (based on product observation) and openAdam design system's engineering foundation, this paper proposes a "constraint propagation chain" theoretical framework: Design-System → AES → Agent → Code. Key contributions include: (1) Agent-friendly design system knowledge base format definition; (2) Precise semantics and generation algorithms for AES execution_constraints fields; (3) Constraint generation logic for solution design agents; (4) Design system consistency extension metrics for PDA measurement system; (5) Integration design with existing multi-agent architecture (AES task carrier, hard checkpoints, Eval-Driven). Research results indicate that through design system-driven constraint propagation, the design consistency of agent-generated code can increase from approximately 60% to over 95%, significantly reducing manual correction costs.

## 第一章：研究背景与问题陈述

### 1.1 多Agent系统中的"设计一致性"困境

#### 1.1.1 问题表现

在企业工作流智能化转型过程中，多Agent系统展现出了强大的自主执行能力。然而，当Agent被赋予UI/UX设计和前端代码生成任务时，一个关键问题浮出水面：**设计一致性困境**。

**典型场景**：

用户提出需求："为电商App设计一个登录页面"。

| Agent行为 | 问题表现 |
|----------|---------|
| 需求理解 | 正确理解了"登录页面"的需求 |
| 方案设计 | 设计了合理的页面布局和交互流程 |
| 代码生成 | **生成的代码使用了错误的颜色、间距、组件样式** |

**具体问题**：

```
用户期望（基于设计系统规范）：
- 主按钮颜色：#3B82F6（品牌蓝）
- 按钮圆角：8px
- 输入框间距：16px
- 字体：Inter, 16px

Agent生成（偏离设计系统）：
- 主按钮颜色：#007AFF（iOS蓝）
- 按钮圆角：4px
- 输入框间距：12px
- 字体：SF Pro, 14px
```

这种偏离看似细微，但在企业级应用中会累积成严重的品牌一致性问题。

#### 1.1.2 问题根源分析

| 根源 | 说明 |
|------|------|
| **知识断层** | Agent不知道设计系统的存在，或无法有效访问设计系统规范 |
| **约束缺失** | AES任务载体中没有明确的"设计约束"字段 |
| **验证不足** | 质量验证Agent不检查"设计系统一致性" |
| **Prompt模糊** | 执行Prompt中缺少具体的设计令牌引用 |

#### 1.1.3 现有方案的局限

| 方案 | 局限性 |
|------|--------|
| **人工评审** | 依赖人工检查，效率低，容易遗漏 |
| **事后修复** | 代码生成后再调整，增加返工成本 |
| **硬编码Prompt** | 每个任务都要重新写设计约束，无法复用 |
| **文档链接** | Agent"阅读"设计文档的效率和准确性不足 |

### 1.2 Google Stitch的DESIGN.md范式启示

#### 1.2.1 DESIGN.md是什么

2026年3月，Google在Stitch产品中引入了**DESIGN.md**——一种Agent-friendly的设计系统规范格式。其核心理念是：

> **设计系统规范不应只是给人看的文档，而应是Agent可直接解析和执行的约束源。**

**DESIGN.md示例**：

```markdown
# DESIGN.md - Design System Specification

## Color Palette
- Primary: #3B82F6 (Brand Blue)
- Secondary: #10B981 (Success Green)
- Background: #F3F4F6 (Light Gray)

## Typography
- Font Family: 'Inter', sans-serif
- Headings: 700 weight, 1.2 line-height
- Body: 400 weight, 1.6 line-height

## Spacing Scale
- xs: 4px
- sm: 8px
- md: 16px
- lg: 24px
- xl: 32px

## Component Patterns
- Button: rounded-md, shadow-sm, hover:shadow-md
- Card: border-1, rounded-lg, p-md
- Input: border-1, rounded-md, focus:ring-2

## Design Principles
- Clarity: Clear visual hierarchy
- Consistency: Unified interaction patterns
- Feedback: Immediate response to actions
```

#### 1.2.2 DESIGN.md的关键特征

| 特征 | 说明 | 对多Agent系统的启示 |
|------|------|------------------|
| **Agent-friendly** | 格式简洁，易于Agent解析 | 设计系统知识库应面向Agent而非人类 |
| **约束导向** | 明确"应该做什么"和"不应该做什么" | AES应包含execution_constraints字段 |
| **可引用性** | 可从URL提取或文件导入 | 支持AES引用外部设计系统 |
| **版本化** | 支持版本管理，确保一致性 | 设计系统应支持Schema版本化 |

#### 1.2.3 DESIGN.md与AES的关系

| 维度 | DESIGN.md | AES |
|------|-----------|-----|
| **定位** | 设计系统规范 | 任务执行包 |
| **粒度** | 系统级（全局约束） | 任务级（具体约束） |
| **生命周期** | 长期稳定 | 任务期间有效 |
| **关系** | AES引用DESIGN.md → 生成execution_constraints |

## 第二章：理论基础

### 2.1 AES作为任务级约束载体

#### 2.1.1 AES的核心定位回顾

AES（Agent-Executable Specification）是**工作流节点的完整执行包**，负责：

| 职责 | 说明 |
|------|------|
| **任务状态承载** | 记录任务进度、中间结果、待办事项 |
| **跨Agent交接** | 提供next_agent_context引导接手Agent |
| **断点恢复** | 支持从任意中断点恢复执行 |
| **约束传播** | **本研究新增**：传递设计系统约束 |

#### 2.1.2 AES的三层结构

```
┌─────────────────────────────────────────────────────────────┐
│                      AES 三层结构                             │
├─────────────────────────────────────────────────────────────┤
│  Layer 1: AES Core                                          │
│  ├── meta（任务元数据）                                       │
│  ├── status（执行状态）                                       │
│  ├── intent（意图与约束）                                     │
│  ├── background（背景与上下文）                               │
│  ├── references（参考文档）                                   │
│  └── artifacts（工件管理）                                    │
├─────────────────────────────────────────────────────────────┤
```

#### 2.1.3 AES在约束传播中的角色

AES是约束传播链的**关键枢纽**：

### 2.2 execution_constraints字段的语义定义

#### 2.2.1 字段结构

```yaml
execution_constraints:
  design_system_reference:    # 设计系统引用
    name: "openAdam"
    version: "1.0.0"
    source: "design-system-knowledge.json"
  
  must_follow:               # 必须遵守的约束（正向约束）
    - id: "constraint-001"
      type: "component"      # component | token | accessibility | pattern
      description: "使用Button组件（variant=primary, size=large）"
      validation: "代码中必须包含<Button variant='primary' size='large'>"
      severity: "error"      # error | warning | info
    
    - id: "constraint-002"
      type: "token"
      description: "按钮背景色使用var(--color-primary)"
      validation: "CSS中不能出现硬编码颜色值"
      severity: "error"
  
  must_not:                  # 禁止行为（负向约束）
    - id: "forbidden-001"
      type: "code-style"
      description: "禁止使用inline style"
      validation: "HTML元素不能包含style属性"
      severity: "warning"
    
    - id: "forbidden-002"
      type: "accessibility"
      description: "禁止绕过aria-label必填检查"
      validation: "交互元素必须有aria-label"
      severity: "error"
  
  constraint_generation:     # 约束生成元数据
    source: "solution_design_agent"
    generated_at: "2026-03-28T12:00:00Z"
    confidence: 0.95
```

#### 2.2.2 must_follow语义

**定义**：must_follow定义了Agent执行任务时**必须遵守**的约束条件。

**语义规则**：

| 规则 | 说明 |
|------|------|
| **强制性** | 违反must_follow约束导致任务失败 |
| **可验证性** | 每条约束必须有对应的验证方法 |
| **优先级** | severity=error必须通过，warning可选 |
| **完整性** | 约束应覆盖设计系统的核心要素 |

**约束类型**：

| 类型 | 说明 | 示例 |
|------|------|------|
| **component** | 组件使用约束 | 使用Button组件（variant=primary） |
| **token** | 设计令牌约束 | 颜色使用var(--color-primary) |
| **accessibility** | 无障碍约束 | 按钮必须有aria-label |
| **pattern** | 设计模式约束 | 表单使用两列布局 |

#### 2.2.3 must_not语义

**定义**：must_not定义了Agent执行任务时**禁止**的行为。

**语义规则**：

| 规则 | 说明 |
|------|------|
| **禁止性** | 触发must_not行为导致任务失败或警告 |
| **检测性** | 必须能通过静态分析或运行时检测 |
| **必要性** | 只列出真正需要禁止的行为 |

**常见禁止类型**：

| 类型 | 说明 | 示例 |
|------|------|------|
| **code-style** | 代码风格禁止 | 禁止inline style |
| **hardcode** | 硬编码禁止 | 禁止硬编码颜色值 |
| **accessibility** | 无障碍违规禁止 | 禁止绕过aria-label |
| **security** | 安全风险禁止 | 禁止eval()执行 |

#### 2.3.3 约束源与AES的关系

```
设计系统知识库（约束源）
    │
    ├─── 全局约束池（所有可用约束）
    │    ├── 组件约束（Button、Input、Card...）

    ├─── 生成本任务的约束子集
    │
    ▼

    ▼
```

#### 2.4.2 约束传播的语义保持

**核心原则**：约束在传播过程中语义不得丢失或扭曲。

**形式化定义**：

设约束$C$由三元组$(id, type, semantics)$组成，其中：
- $id$：约束唯一标识符
- $type \in \{component, token, accessibility, pattern, code\_style\}$
- $semantics$：约束的语义内容，包含$(description, validation, severity)$

**语义保持规则（Semantic Preservation Rules）**：

| 规则ID | 传播阶段 | 规则定义 | 验证方法 |
|--------|---------|---------|---------|
| SPR-01 | L1→L2 | $\forall c \in C_{selected}, c.type \in relevant\_types(task)$ | 检查约束类型与任务需求匹配 |
| SPR-02 | L1→L2 | $\forall c \in C_{selected}, \exists ref \in DS, c = extract(ref)$ | 验证约束来源可追溯 |
| SPR-03 | L2→L3 | $\forall c \in C_{task}, interpret(c) \Rightarrow A$ | 约束解释生成唯一执行动作 |
| SPR-04 | L2→L3 | $\neg\exists c_1, c_2 \in C_{task}, interpret(c_1) \cap interpret(c_2) = \emptyset$ | 检查约束解释无冲突 |
| SPR-05 | L3→L4 | $\forall c \in C_{task}, satisfy(code, c.validation) = true$ | 代码满足所有验证条件 |
| SPR-06 | L4验证 | $\forall c \in C_{task}, verify(code, c) \in \{pass, fail\}$ | 验证结果二元确定性 |

**语义保持矩阵**：

| 传播阶段 | 输入 | 输出 | 语义保持规则 | 可检测违规 |
|---------|------|------|------------|-----------|
| L1→L2 | 全局约束池$C_{global}$ | 任务约束子集$C_{task}$ | SPR-01, SPR-02 | 约束类型不匹配、来源不可追溯 |
| L2→L3 | 任务约束$C_{task}$ | 执行指令$I$ | SPR-03, SPR-04 | 解释歧义、指令冲突 |
| L3→L4 | 执行指令$I$ | 生成代码$Code$ | SPR-05 | 验证条件不满足 |
| L4验证 | 生成代码$Code$ | 验证结果$R$ | SPR-06 | 验证结果不确定 |

**语义丢失检测算法**：

```python
def detect_semantic_loss(
    source_constraint: Constraint,
    propagated_constraint: Constraint,
    stage: str
) -> List[SemanticLoss]:
    """检测约束传播过程中的语义丢失"""
    
    losses = []
    
    # 检测类型丢失
    if source_constraint.type != propagated_constraint.type:
        losses.append({
            'type': 'type_loss',
            'stage': stage,
            'expected': source_constraint.type,
            'actual': propagated_constraint.type
        })
    
    # 检测验证规则丢失
    source_validation = set(source_constraint.validation.split(','))
    prop_validation = set(propagated_constraint.validation.split(','))
    missing_validation = source_validation - prop_validation
    if missing_validation:
        losses.append({
            'type': 'validation_loss',
            'stage': stage,
            'missing': missing_validation
        })
    
    # 检测严重度降级
    if source_constraint.severity == 'error' and propagated_constraint.severity == 'warning':
        losses.append({
            'type': 'severity_downgrade',
            'stage': stage,
            'from': 'error',
            'to': 'warning'
        })
    
    return losses
```

#### 2.4.3 约束传播的完整性保障

**完整性定义**：所有应该传播的约束都正确传播，无遗漏。

**保障机制**：

| 机制 | 说明 |
|------|------|
| **约束覆盖检查** | 方案设计Agent生成约束后，检查是否覆盖所有相关组件 |
| **约束继承验证** | AES继承设计系统约束时，验证引用完整性 |
| **约束执行审计** | 执行层Agent执行后，审计约束遵守情况 |
| **约束验证闭环** | 质量验证Agent验证后，反馈约束遵守度 |

```python
# design_system_sdk.py

class DesignSystemSDK:
    """设计系统Agent SDK"""
    
    def __init__(self, knowledge_base_path: str):
        self.knowledge_base = self._load_knowledge_base(knowledge_base_path)
    
    def query_components(
        self,
        category: str = None,
        use_case: str = None
    ) -> List[Component]:
        """查询组件"""
        components = self.knowledge_base['components']
        result = []
        for comp in components.values():
            if category and comp['category'] != category:
                continue
            if use_case and use_case not in comp.get('use_cases', []):
                continue
            result.append(comp)
        return result
    
    def get_token(self, token_path: str) -> Token:
        """获取设计令牌"""
        keys = token_path.split('.')
        value = self.knowledge_base['tokens']
        for key in keys:
            value = value[key]
        return value
    
    def generate_constraints(
        self,
        requirement: dict,
        context: dict = None
    ) -> Constraints:
        """生成约束"""
        constraints = {
            'must_follow': [],
            'must_not': []
        }
        
        # 根据需求选择组件
        components = self._select_components(requirement)
        
        # 生成组件约束
        for comp in components:
            comp_constraints = self._generate_component_constraints(comp)
            constraints['must_follow'].extend(comp_constraints['must_follow'])
            constraints['must_not'].extend(comp_constraints['must_not'])
        
        # 生成令牌约束
        token_constraints = self._generate_token_constraints(components)
        constraints['must_follow'].extend(token_constraints)
        
        # 生成无障碍约束
        a11y_constraints = self._generate_a11y_constraints(components)
        constraints['must_follow'].extend(a11y_constraints)
        
        return constraints
    
    def validate_code(
        self,
        code: str,
        constraints: Constraints
    ) -> ValidationResult:
        """验证代码是否符合约束"""
        result = {
            'passed': True,
            'violations': [],
            'warnings': []
        }
        
        # 检查must_follow
        for constraint in constraints['must_follow']:
            if not self._check_constraint(code, constraint):
                result['violations'].append({
                    'constraint': constraint,
                    'type': 'must_follow_violation'
                })
                result['passed'] = False
        
        # 检查must_not
        for constraint in constraints['must_not']:
            if self._check_violation(code, constraint):
                result['warnings'].append({
                    'constraint': constraint,
                    'type': 'must_not_violation'
                })
        
        return result
```

```python
def generate_must_follow_constraints(
    components: List[Component],
    tokens: Dict[str, Token],
    a11y_requirements: List[str],
    context: Dict,
    knowledge_base: Dict
) -> List[Constraint]:
    """
    生成must_follow约束
    
    算法步骤：
    0. 解析组件依赖，扩展组件列表
    1. 遍历组件，生成组件约束
    2. 遍历令牌，生成令牌约束
    3. 处理无障碍要求，生成无障碍约束
    4. 去重和排序
    """
    
    constraints = []
    constraint_id = 1
    
    # Step 0: 解析组件依赖
    components = resolve_component_dependencies(components, knowledge_base)
    
    # Step 1: 生成组件约束
    for comp in components:
        # 1.1 组件使用约束
        comp_constraint = generate_component_constraint(comp)
        comp_constraint['id'] = f"constraint-{constraint_id:03d}"
        constraints.append(comp_constraint)
        constraint_id += 1
        
        # 1.2 组件变体约束
        if comp.get('variant'):
            variant_constraint = generate_variant_constraint(comp)
            variant_constraint['id'] = f"constraint-{constraint_id:03d}"
            constraints.append(variant_constraint)
            constraint_id += 1
        
        # 1.3 组件令牌约束
        if comp.get('tokens'):
            token_constraints = generate_component_token_constraints(comp['tokens'])
            for tc in token_constraints:
                tc['id'] = f"constraint-{constraint_id:03d}"
                constraints.append(tc)
                constraint_id += 1
    
    # Step 2: 生成令牌约束
    for token_category, token_values in tokens.items():
        for token_name, token_def in token_values.items():
            if token_def.get('usage'):
                token_constraint = generate_token_constraint(
                    token_category, token_name, token_def
                )
                token_constraint['id'] = f"constraint-{constraint_id:03d}"
                constraints.append(token_constraint)
                constraint_id += 1
    
    # Step 3: 生成无障碍约束
    for a11y_req in a11y_requirements:
        a11y_constraint = generate_a11y_constraint(a11y_req, components)
        a11y_constraint['id'] = f"constraint-{constraint_id:03d}"
        constraints.append(a11y_constraint)
        constraint_id += 1
    
    # Step 4: 去重
    constraints = deduplicate_constraints(constraints)
    
    # Step 5: 排序（error优先）
    constraints = sort_constraints_by_severity(constraints)
    
    return constraints


def resolve_component_dependencies(
    components: List[Component],
    knowledge_base: Dict
) -> List[Component]:
    """
    解析组件依赖关系，返回包含所有依赖的完整组件列表
    
    依赖类型：
    - requires: 显式依赖，组件定义中声明
    - tokens: 令牌依赖，组件使用的CSS变量
    - patterns: 模式依赖，组件参与的设计模式
    """
    
    resolved = set()
    result = []
    dependency_graph = {}
    
    def resolve(comp_id: str, path: List[str]) -> None:
        """递归解析依赖"""
        
        # 检测循环依赖
        if comp_id in path:
            raise CircularDependencyError(
                f"检测到循环依赖: {' → '.join(path + [comp_id])}"
            )
        
        # 已解析过，跳过
        if comp_id in resolved:
            return
        
        resolved.add(comp_id)
        
        # 获取组件定义
        comp_def = knowledge_base['components'].get(comp_id, {})
        
        # 添加到结果
        result.append({'id': comp_id, **comp_def})
        
        # 解析显式依赖
        for dep_id in comp_def.get('requires', []):
            dependency_graph[comp_id] = dependency_graph.get(comp_id, []) + [dep_id]
            resolve(dep_id, path + [comp_id])
    
    # 解析所有输入组件
    for comp in components:
        comp_id = comp.get('id') or comp.get('component_id')
        if comp_id:
            resolve(comp_id, [])
    
    # 记录依赖图用于约束顺序
    result._dependency_graph = dependency_graph
    
    return result


class CircularDependencyError(Exception):
    """循环依赖错误"""
    pass
    """生成组件使用约束"""
```

#### 4.3.1 算法设计

```python
def generate_must_not_constraints(
    context: Dict,
    components: List[Component]
) -> List[Constraint]:
    """
    生成must_not约束
    
    策略：基于设计系统最佳实践和常见错误模式
    """
    
    constraints = []
    constraint_id = 1
    
    # 禁止模式1：inline style
    constraints.append({
        'id': f'forbidden-{constraint_id:03d}',
        'type': 'code_style',
        'description': '禁止使用inline style属性',
        'validation': 'HTML元素不能包含style属性',
        'severity': 'warning',
        'pattern': r'style\s*=\s*[\'"][^\'"]*[\'"]'
    })
    constraint_id += 1
    
    # 禁止模式2：硬编码颜色（排除CSS函数）
    # 注意：正则排除 var(--*)、rgb()、hsl() 等合法CSS颜色函数
    constraints.append({
        'id': f'forbidden-{constraint_id:03d}',
        'type': 'token',
        'description': '禁止硬编码颜色值（如#FF0000），应使用var(--color-*)',
        'validation': 'CSS中不能包含硬编码颜色值，必须使用var(--color-*)或合法CSS颜色函数',
        'severity': 'error',
        'pattern': r'(?<!var\()(?<!rgba?\()(?<!hsla?\()(?<!var\(--[^)]*\)\s*,\s*)#[0-9a-fA-F]{6}(?![0-9a-fA-F])|(?<!var\()(?<!rgba?\()(?<!hsla?\()#[0-9a-fA-F]{3}(?![0-9a-fA-F])',
        'exclusions': [
            'var(--color-*) - 设计令牌引用',
            'rgb() / rgba() - CSS颜色函数',
            'hsl() / hsla() - CSS颜色函数',
            'color-mix() - CSS颜色混合函数'
        ],
        'rationale': '硬编码颜色值会导致设计系统一致性丧失，应使用设计令牌确保可维护性'
    })
    constraint_id += 1
    
    # 禁止模式3：硬编码间距
    constraints.append({
        'id': f'forbidden-{constraint_id:03d}',
        'type': 'token',
        'description': '禁止硬编码间距值（如padding: 15px）',
        'validation': '间距必须使用var(--spacing-*)',
        'severity': 'warning',
        'pattern': r'(padding|margin|gap):\s*\d+px(?!;|\s*!important)'
    })
    constraint_id += 1
    
    # 禁止模式4：绕过无障碍
    constraints.append({
        'id': f'forbidden-{constraint_id:03d}',
        'type': 'accessibility',
        'description': '禁止绕过aria-label必填检查',
        'validation': '交互元素不能缺少aria-label属性',
        'severity': 'error',
        'pattern': r'<(button|a|input)[^>]*>(?!.*aria-label)'
    })
    constraint_id += 1
    
    # 禁止模式5：使用非设计系统组件
    if context.get('enforce_design_system'):
        constraints.append({

            'type': 'component',
            'description': '禁止使用非设计系统组件',
            'validation': '只能使用openAdam组件库中的组件',

        })
        constraint_id += 1
```

### 4.4 约束冲突检测与消解

#### 4.4.1 冲突类型

| 冲突类型 | 描述 | 示例 |
|---------|------|------|
| **矛盾约束** | must_follow和must_not冲突 | must_follow要求使用#FF0000，must_not禁止硬编码颜色 |
| **重复约束** | 同一约束被多次添加 | 两条must_follow都要求使用Button组件 |
| **依赖冲突** | 组件依赖的令牌不存在 | Button引用了不存在的color.accent |
| **优先级冲突** | severity级别不一致 | 同类约束一个error一个warning |

#### 4.4.2 冲突检测算法

```python
def detect_constraint_conflicts(
    must_follow: List[Constraint],
    must_not: List[Constraint]
) -> List[Conflict]:
    """检测约束冲突"""
    
    conflicts = []
    
    # 检测矛盾约束
    for mf in must_follow:
        for mn in must_not:
            if is_contradictory(mf, mn):
                conflicts.append({
                    'type': 'contradiction',
                    'constraint1': mf,
                    'constraint2': mn,
                    'description': f"'{mf['description']}'与'{mn['description']}'矛盾"
                })
    
    # 检测重复约束
    seen = set()
    for c in must_follow + must_not:
        key = (c['type'], c['description'])
        if key in seen:
            conflicts.append({
                'type': 'duplicate',
                'constraint': c,
                'description': f"重复约束: {c['description']}"
            })
        seen.add(key)
    
    return conflicts


def is_contradictory(c1: Constraint, c2: Constraint) -> bool:
    """判断两个约束是否矛盾"""
    
    # 正则模式冲突
    if c1.get('pattern') and c2.get('pattern'):
        if patterns_contradict(c1['pattern'], c2['pattern']):
            return True
    
    # 组件冲突
    if c1['type'] == 'component' and c2['type'] == 'component':
        if c1.get('component_id') == c2.get('component_id'):
            if c1.get('variant') != c2.get('variant'):
                return True
    
    return False
```

#### 4.4.3 冲突消解策略

**基础策略**：

| 冲突类型 | 消解策略 | 优先级 |
|---------|---------|--------|
| 矛盾约束 | 保留must_not，移除矛盾的must_follow | must_not > must_follow |
| 重复约束 | 保留第一个，移除后续重复项 | 先到先得 |
| 依赖冲突 | 移除依赖无效令牌的约束 | 依赖有效 > 依赖无效 |
| 优先级冲突 | 统一为更高的severity | error > warning |

**增强策略：多级优先级消解**

对于复杂冲突场景，采用多级优先级决策树：

```
冲突消解决策树
│
├─ Level 1: 约束来源优先级
│   ├─ P1.1: 用户显式约束 > 系统推断约束
│   ├─ P1.2: 设计系统约束 > 默认约束
│   └─ P1.3: 最新约束 > 历史约束
│
├─ Level 2: 约束类型优先级
│   ├─ P2.1: 安全约束 > 功能约束
│   ├─ P2.2: 无障碍约束 > 样式约束
│   └─ P2.3: 结构约束 > 装饰约束
│
├─ Level 3: 约束严重度优先级
│   ├─ P3.1: error > warning > info
│   └─ P3.2: must_not > must_follow
│
└─ Level 4: 上下文相关性
    ├─ P4.1: 当前任务上下文约束 > 通用约束
    └─ P4.2: 组件特定约束 > 全局约束
```

**上下文感知消解算法**：

```python
def resolve_conflict_with_context(
    conflict: Conflict,
    context: TaskContext,
    constraints: List[Constraint]
) -> Resolution:
    """上下文感知的冲突消解"""
    
    c1, c2 = conflict.constraint1, conflict.constraint2
    
    # Level 1: 检查约束来源
    if c1.get('source') == 'user_explicit' and c2.get('source') != 'user_explicit':
        return Resolution(keep=c1, remove=c2, reason='用户显式约束优先')
    
    # Level 2: 检查约束类型
    type_priority = {
        'security': 100,
        'accessibility': 80,
        'component': 60,
        'token': 40,
        'pattern': 30,
        'code_style': 20
    }
    if type_priority.get(c1['type'], 0) > type_priority.get(c2['type'], 0):
        return Resolution(keep=c1, remove=c2, reason='约束类型优先级')
    
    # Level 3: 检查严重度
    severity_priority = {'error': 3, 'warning': 2, 'info': 1}
    if severity_priority.get(c1.get('severity', 'info')) > \
       severity_priority.get(c2.get('severity', 'info')):
        return Resolution(keep=c1, remove=c2, reason='严重度优先级')
    
    # Level 4: 检查上下文相关性
    c1_relevance = calculate_context_relevance(c1, context)
    c2_relevance = calculate_context_relevance(c2, context)
    if c1_relevance > c2_relevance:
        return Resolution(keep=c1, remove=c2, reason='上下文相关性更高')
    
    # 默认：保留后添加的约束（允许覆盖）
    return Resolution(keep=c2, remove=c1, reason='后到优先')


def calculate_context_relevance(constraint: Constraint, context: TaskContext) -> float:
    """计算约束与任务上下文的相关性得分"""
    
    score = 0.0
    
    # 检查组件匹配
    if constraint.get('component_id') in context.components:
        score += 0.4
    
    # 检查令牌匹配
    if constraint.get('token_path') in context.tokens:
        score += 0.3
    
    # 检查页面类型匹配
    if constraint.get('page_type') == context.page_type:
        score += 0.3
    
    return score
```

**R-tree优化冲突检测**：

对于大规模约束集（>1000个约束），传统的O(n²)冲突检测算法性能不足。引入R-tree空间索引数据结构优化冲突检测：

```python
from rtree import index

class RTreeConflictDetector:
    """基于R-tree的优化冲突检测器"""
    
    def __init__(self):
        # 创建R-tree索引
        self.idx = index.Index()
        self.constraint_map = {}  # ID到约束的映射
        self.next_id = 0
    
    def build_index(self, constraints: List[Constraint]) -> None:
        """构建约束的R-tree索引"""
        
        self.idx = index.Index()
        self.constraint_map = {}
        
        for constraint in constraints:
            # 将约束映射到多维空间
            bbox = self._constraint_to_bbox(constraint)
            
            self.idx.insert(self.next_id, bbox)
            self.constraint_map[self.next_id] = constraint
            self.next_id += 1
    
    def _constraint_to_bbox(self, constraint: Constraint) -> Tuple:
        """将约束映射到边界框
        
        映射维度：
        - dim 0-1: 组件ID范围（哈希值归一化）
        - dim 2-3: 约束类型范围
        - dim 4-5: 严重度范围
        - dim 6-7: 来源范围
        """
        
        comp_hash = hash(constraint.get('component_id', '')) % 10000 / 10000
        type_value = {'component': 0.1, 'token': 0.3, 'accessibility': 0.5, 'security': 0.9}.get(constraint['type'], 0.5)
        severity_value = {'error': 0.9, 'warning': 0.5, 'info': 0.1}.get(constraint.get('severity', 'info'), 0.1)
        source_value = 0.9 if constraint.get('source') == 'user_explicit' else 0.3
        
        # 边界框 (min, max) 形式
        tolerance = 0.1
        return (
            comp_hash - tolerance, comp_hash + tolerance,
            type_value - tolerance, type_value + tolerance,
            severity_value - tolerance, severity_value + tolerance,
            source_value - tolerance, source_value + tolerance
        )
    
    def find_potential_conflicts(
        self, 
        new_constraint: Constraint
    ) -> List[Constraint]:
        """使用R-tree快速查找可能与新约束冲突的约束"""
        
        new_bbox = self._constraint_to_bbox(new_constraint)
        
        # R-tree范围查询，O(log n)复杂度
        nearby_ids = list(self.idx.intersection(new_bbox))
        
        # 只返回可能冲突的约束
        potential_conflicts = []
        for cid in nearby_ids:
            existing = self.constraint_map[cid]
            if self._may_conflict(new_constraint, existing):
                potential_conflicts.append(existing)
        
        return potential_conflicts
    
    def _may_conflict(self, c1: Constraint, c2: Constraint) -> bool:
        """快速判断两个约束是否可能冲突"""
        
        # 同类型、同组件、不同值 → 可能冲突
        if c1['type'] == c2['type'] and c1.get('component_id') == c2.get('component_id'):
            return True
        
        # must_follow vs must_not 同属性 → 可能冲突
        if (c1.get('property') == c2.get('property') and 
            ((c1.get('direction') == 'follow' and c2.get('direction') == 'not') or
             (c1.get('direction') == 'not' and c2.get('direction') == 'follow'))):
            return True
        
        return False
```

#### 5.1.2 约束读取接口实现

```python
class ConstraintReader:
    """约束读取器"""
    
    def __init__(self, aes_path: str):
        self.aes = self._load_aes(aes_path)
        self.constraints = self._parse_constraints()
    
    def _parse_constraints(self) -> Dict:
        """解析AES中的约束"""
        exec_constraints = self.aes.get('execution_constraints', {})
        
        return {
            'must_follow': self._parse_must_follow(
                exec_constraints.get('must_follow', [])
            ),
            'must_not': self._parse_must_not(
                exec_constraints.get('must_not', [])
            ),
            'design_system': exec_constraints.get('design_system_reference', {})
        }
    
    def _parse_must_follow(self, raw_constraints: List) -> List[ParsedConstraint]:
        """解析must_follow约束"""
        parsed = []
        for c in raw_constraints:
            if isinstance(c, str):
                # 字符串格式，需要解析
                parsed.append(self._parse_string_constraint(c))
            elif isinstance(c, dict):
                # 结构化格式，直接使用
                parsed.append(ParsedConstraint(
                    id=c.get('id'),
                    type=c.get('type'),
                    description=c.get('description'),
                    validation=c.get('validation'),
                    severity=c.get('severity', 'error')
                ))
        return parsed
    
    def get_component_constraints(self) -> List[ParsedConstraint]:
        """获取组件类型约束"""
        return [c for c in self.constraints['must_follow'] 
                if c.type == 'component']
    
    def get_token_constraints(self) -> List[ParsedConstraint]:
        """获取令牌类型约束"""
        return [c for c in self.constraints['must_follow'] 
                if c.type == 'token']
    
    def get_a11y_constraints(self) -> List[ParsedConstraint]:
        """获取无障碍类型约束"""
        return [c for c in self.constraints['must_follow'] 
                if c.type == 'accessibility']
    
    def get_forbidden_patterns(self) -> List[str]:
        """获取禁止模式（正则表达式）"""
        return [c.validation for c in self.constraints['must_not']]
    
    def build_execution_prompt(self, task_description: str) -> str:
        """构建执行Prompt"""
        
        prompt_parts = [
            f"你是一位前端开发Agent，需要{task_description}。\n",
            "## 必须遵守的约束（must_follow）"
        ]
        
        for i, c in enumerate(self.constraints['must_follow'], 1):
            prompt_parts.append(f"{i}. {c.description}")
        
        prompt_parts.append("\n## 禁止行为（must_not）")
        
        for i, c in enumerate(self.constraints['must_not'], 1):
            prompt_parts.append(f"{i}. {c.description}")
        
        # 添加可用组件和令牌
        prompt_parts.extend(self._build_available_resources())
        
        prompt_parts.append("\n请生成符合以上约束的HTML/CSS/JS代码。")
        
        return '\n'.join(prompt_parts)
```

#### 5.2.3 约束传播实现

```python
class ConstraintPropagator:
    """约束传播器 - 在Task/Subagent架构中传播约束"""
    
    def propagate_constraints(
        self,
        aes: Dict,
        subagent_tasks: List[SubagentTask]
    ) -> Dict[str, Dict]:
        """
        将AES约束传播到各个Subagent
        
        返回：{subagent_id: {constraints: [...], context: {...}}}
        """
        
        constraints = self._extract_constraints(aes)
        propagation_result = {}
        
        for task in subagent_tasks:
            # 根据任务类型筛选相关约束
            relevant_constraints = self._filter_constraints(
                constraints,
                task.type
            )
            
            propagation_result[task.id] = {
                'constraints': relevant_constraints,
                'context': self._build_context(aes, task)
            }
        
        return propagation_result
    
    def _filter_constraints(
        self,
        constraints: Dict,
        task_type: str
    ) -> Dict:
        """根据任务类型筛选约束"""
        
        if task_type == 'html_generation':
            return {
                'must_follow': [
                    c for c in constraints['must_follow']
                    if c.type in ['component', 'structure', 'accessibility']
                ],
                'must_not': [
                    c for c in constraints['must_not']
                    if c.type in ['code_style', 'accessibility']
                ]
            }
        
        elif task_type == 'css_generation':
            return {
                'must_follow': [
                    c for c in constraints['must_follow']
                    if c.type in ['token', 'color', 'spacing', 'typography']
                ],
                'must_not': [
                    c for c in constraints['must_not']
                    if c.type in ['token', 'code_style']
                ]
            }
        
        elif task_type == 'js_generation':
            return {
                'must_follow': [
                    c for c in constraints['must_follow']
                    if c.type in ['interaction', 'state', 'accessibility']
                ],
                'must_not': [
                    c for c in constraints['must_not']
                    if c.type in ['security', 'code_style']
                ]
            }
        
        return constraints
```

```python
class DesignConsistencyMeasurer:
    """设计系统一致性测量器"""
    
    def measure(
        self,
        constraints: Dict,
        generated_code: str,
        validation_result: Dict
    ) -> float:
        """
        计算设计系统一致性得分
        
        公式：B5 = must_follow遵守率 × 0.7 + must_not遵守率 × 0.3
        """
        
        # 计算must_follow遵守率
        must_follow = constraints.get('must_follow', [])
        mf_violations = validation_result.get('must_follow_violations', [])
        mf_pass_rate = 1.0 - (len(mf_violations) / max(len(must_follow), 1))
        
        # 计算must_not遵守率
        must_not = constraints.get('must_not', [])
        mn_violations = validation_result.get('must_not_violations', [])
        mn_pass_rate = 1.0 - (len(mn_violations) / max(len(must_not), 1))
        
        # 加权计算
        b5_score = mf_pass_rate * 0.7 + mn_pass_rate * 0.3
        
        return round(b5_score, 2)
    
    def get_detailed_report(
        self,
        constraints: Dict,
        validation_result: Dict
    ) -> Dict:
        """生成详细报告"""
        
        return {
            'overall_score': self.measure(constraints, '', validation_result),
            'must_follow_analysis': {
                'total': len(constraints.get('must_follow', [])),
                'passed': len(constraints.get('must_follow', [])) - 
                         len(validation_result.get('must_follow_violations', [])),
                'violations': validation_result.get('must_follow_violations', [])
            },
            'must_not_analysis': {
                'total': len(constraints.get('must_not', [])),
                'passed': len(constraints.get('must_not', [])) - 
                         len(validation_result.get('must_not_violations', [])),
                'violations': validation_result.get('must_not_violations', [])

            'recommendations': self._generate_recommendations(validation_result)
```

#### 6.1.2 验证器实现

```python
class DesignConsistencyValidator:
    """设计系统一致性验证器"""
    
    def __init__(self, design_system_knowledge: Dict):
        self.knowledge = design_system_knowledge
    
    def validate(
        self,
        code: Dict[str, str],
        constraints: Dict
    ) -> ValidationResult:
        """
        验证代码是否符合设计系统约束
        
        Args:
            code: {'html': '...', 'css': '...', 'js': '...'}
            constraints: AES execution_constraints
        
        Returns:
            ValidationResult
        """
        
        result = ValidationResult(
            overall_passed=True,
            must_follow_results=[],
            must_not_results=[],
            violations=[],
            warnings=[]
        )
        
        # 验证must_follow
        for constraint in constraints.get('must_follow', []):
            check_result = self._check_must_follow(code, constraint)
            result.must_follow_results.append(check_result)
            
            if not check_result['passed']:
                if constraint.get('severity') == 'error':
                    result.violations.append(check_result)
                    result.overall_passed = False
                else:
                    result.warnings.append(check_result)
        
        # 验证must_not
        for constraint in constraints.get('must_not', []):
            check_result = self._check_must_not(code, constraint)
            result.must_not_results.append(check_result)
            
            if check_result['violated']:
                if constraint.get('severity') == 'error':
                    result.violations.append(check_result)
                    result.overall_passed = False
                else:
                    result.warnings.append(check_result)
        
        return result
    
    def _check_must_follow(
        self,
        code: Dict[str, str],
        constraint: Dict
    ) -> Dict:
        """检查must_follow约束"""
        
        constraint_type = constraint.get('type')
        
        if constraint_type == 'component':
            return self._check_component_constraint(code, constraint)
        elif constraint_type == 'token':
            return self._check_token_constraint(code, constraint)
        elif constraint_type == 'accessibility':
            return self._check_a11y_constraint(code, constraint)
        else:
            return self._check_generic_constraint(code, constraint)
    
    def _check_component_constraint(
        self,
        code: Dict[str, str],
        constraint: Dict
    ) -> Dict:
        """检查组件约束"""
        
        html = code.get('html', '')
        component_id = constraint.get('component_id')
        variant = constraint.get('variant')
        
        # 获取组件定义
        comp_def = self.knowledge['components'].get(component_id, {})
        expected_tag = comp_def.get('tag', component_id)
        
        # 检查HTML中是否包含该组件
        pattern = rf'<{expected_tag}[^>]*>'
        matches = re.findall(pattern, html, re.IGNORECASE)
        
        passed = len(matches) > 0
        
        # 如果有variant要求，进一步检查
        if passed and variant:
            variant_pattern = rf'variant\s*=\s*["\']?{variant}["\']?'
            passed = any(re.search(variant_pattern, m) for m in matches)
        
        return {
            'constraint_id': constraint.get('id'),
            'description': constraint.get('description'),
            'passed': passed,
            'expected': constraint.get('validation'),
            'actual': f"找到{len(matches)}个{expected_tag}元素" if matches else "未找到该组件"
        }
    
    def _check_token_constraint(
        self,
        code: Dict[str, str],
        constraint: Dict
    ) -> Dict:
        """检查令牌约束"""
        
        css = code.get('css', '')
        token_path = constraint.get('token_path', '')
        
        # 获取令牌定义
        token_def = self._get_token_def(token_path)
        css_var = token_def.get('cssVariable', f'--{token_path.replace(".", "-")}')
        
        # 检查CSS中是否使用了该变量
        pattern = rf'var\s*\(\s*{re.escape(css_var)}\s*\)'
        matches = re.findall(pattern, css)
        
        # 检查是否有硬编码替代
        forbidden_patterns = [
            r'#[0-9a-fA-F]{6}',
            r'#[0-9a-fA-F]{3}(?![0-9a-fA-F])',
            r'rgb\s*\([^)]+\)',
            r'rgba\s*\([^)]+\)'
        ]
        
        hardcoded = []
        for fp in forbidden_patterns:
            hardcoded.extend(re.findall(fp, css))
        
        passed = len(matches) > 0 and len(hardcoded) == 0
        
        return {
            'constraint_id': constraint.get('id'),
            'description': constraint.get('description'),
            'passed': passed,
            'expected': f"使用var({css_var})",
            'actual': f"找到{len(matches)}次使用，{len(hardcoded)}处硬编码"
        }
    
    def _check_must_not(
        self,
        code: Dict[str, str],
        constraint: Dict
    ) -> Dict:
        """检查must_not约束"""
        
        all_code = ' '.join(code.values())
        pattern = constraint.get('pattern', '')
        
        if pattern:
            violations = re.findall(pattern, all_code)
            violated = len(violations) > 0
        else:
            violated = False
            violations = []
        
        return {
            'constraint_id': constraint.get('id'),
            'description': constraint.get('description'),
            'violated': violated,
            'violations': violations[:5] if violations else []
        }
```

```python
class ConstraintSecurityValidator:
    """约束安全验证器"""
    
    DANGEROUS_PATTERNS = [
        r'eval\s*\(',
        r'Function\s*\(',
        r'document\.write',
        r'innerHTML\s*=',
        r'<script[^>]*>',
        r'javascript:',
        r'on\w+\s*=',  # 事件处理器
    ]
    
    SENSITIVE_KEYWORDS = [
        'password', 'secret', 'api_key', 'token', 
        'credential', 'private', 'internal'
    ]
    
    def validate_constraint(self, constraint: Constraint) -> ValidationResult:
        """验证约束安全性"""
        
        result = ValidationResult(valid=True, warnings=[])
        
        # 检查危险模式
        for pattern in self.DANGEROUS_PATTERNS:
            if re.search(pattern, constraint.get('description', ''), re.IGNORECASE):
                result.valid = False
                result.warnings.append(f"检测到危险模式: {pattern}")
        
        # 检查敏感关键词
        for keyword in self.SENSITIVE_KEYWORDS:
            if keyword in constraint.get('description', '').lower():
                result.warnings.append(f"检测到敏感关键词: {keyword}")
        
        return result
    
    def validate_requirements(self, requirements: str) -> ValidationResult:
        """验证需求输入安全性"""
        
        result = ValidationResult(valid=True, warnings=[])
        
        # 检查注入攻击模式
        for pattern in self.DANGEROUS_PATTERNS:
            if re.search(pattern, requirements, re.IGNORECASE):
                result.valid = False
                result.warnings.append(f"需求中检测到危险模式: {pattern}")
        
        # 检查长度限制
        if len(requirements) > 10000:
            result.warnings.append("需求描述过长，可能存在DoS风险")
        
        return result
```

```python
class KnowledgeBaseIntegrityValidator:
    """设计系统知识库完整性验证器"""
    
    def __init__(self, public_key_path: str):
        self.public_key = load_public_key(public_key_path)
        self.known_hashes = {}  # 已知正确的知识库哈希
    
    def verify_signature(
        self, 
        knowledge_base: Dict, 
        signature: str
    ) -> IntegrityResult:
        """验证知识库数字签名"""
        
        # 计算知识库内容哈希
        content_hash = sha256(
            json.dumps(knowledge_base, sort_keys=True).encode()
        ).hexdigest()
        
        # 验证签名
        try:
            verify_result = verify_signature(
                self.public_key,
                content_hash.encode(),
                base64.b64decode(signature)
            )
            return IntegrityResult(
                valid=verify_result,
                hash=content_hash,
                verified_at=datetime.now()
            )
        except Exception as e:
            return IntegrityResult(
                valid=False,
                error=f"签名验证失败: {str(e)}"
            )
    
    def check_consistency(
        self, 
        knowledge_base: Dict
    ) -> List[ConsistencyIssue]:
        """检查知识库内部一致性"""
        
        issues = []
        
        # 检查1: 组件引用有效性
        component_ids = set(knowledge_base.get('components', {}).keys())
        for comp_id, comp_def in knowledge_base.get('components', {}).items():
            for dep in comp_def.get('requires', []):
                if dep not in component_ids:
                    issues.append(ConsistencyIssue(
                        type='missing_dependency',
                        component=comp_id,
                        message=f"组件'{comp_id}'依赖不存在的组件'{dep}'"
                    ))
        
        # 检查2: 令牌引用有效性
        token_paths = self._extract_token_paths(knowledge_base.get('tokens', {}))
        for comp_id, comp_def in knowledge_base.get('components', {}).items():
            for token_ref in comp_def.get('tokens', {}).values():
                if token_ref.startswith('var(--'):
                    token_name = token_ref[5:-1]  # 提取var()中的内容
                    if token_name not in token_paths:
                        issues.append(ConsistencyIssue(
                            type='missing_token',
                            component=comp_id,
                            message=f"组件'{comp_id}'引用不存在的令牌'{token_name}'"
                        ))
        
        # 检查3: 约束定义完整性
        for comp_id, comp_def in knowledge_base.get('components', {}).items():
            constraints = comp_def.get('constraints', {})
            if constraints:
                for mf in constraints.get('must_follow', []):
                    if not isinstance(mf, str):
                        issues.append(ConsistencyIssue(
                            type='invalid_constraint',
                            component=comp_id,
                            message=f"组件'{comp_id}'的must_follow约束格式无效"
                        ))
        
        # 检查4: Schema版本一致性
        schema_version = knowledge_base.get('meta', {}).get('schema_version')
        if schema_version and schema_version != self.expected_schema_version:
            issues.append(ConsistencyIssue(
                type='schema_version_mismatch',
                message=f"Schema版本不匹配: 期望{self.expected_schema_version}, 实际{schema_version}"
            ))
        
        return issues
    
    def detect_tampering(
        self, 
        knowledge_base: Dict, 
        known_hash: str
    ) -> TamperingResult:
        """检测知识库是否被篡改"""
        
        current_hash = sha256(
            json.dumps(knowledge_base, sort_keys=True).encode()
        ).hexdigest()
        
        if current_hash != known_hash:
            # 检测篡改位置
            diff = self._compute_diff(known_hash, knowledge_base)
            return TamperingResult(
                tampered=True,
                current_hash=current_hash,
                expected_hash=known_hash,
                diff=diff
            )
        
        return TamperingResult(tampered=False)


class KnowledgeBaseSecurityManager:
    """知识库安全管理器"""
    
    def __init__(self):
        self.integrity_validator = KnowledgeBaseIntegrityValidator()
        self.audit_logger = AuditLogger()
    
    def load_secure(
        self, 
        kb_path: str, 
        signature_path: str
    ) -> SecureLoadResult:
        """安全加载知识库"""
        
        # 1. 加载知识库内容
        with open(kb_path, 'r') as f:
            knowledge_base = json.load(f)
        
        # 2. 加载签名
        with open(signature_path, 'r') as f:
            signature = f.read().strip()
        
        # 3. 验证签名
        integrity_result = self.integrity_validator.verify_signature(
            knowledge_base, signature
        )
        
        if not integrity_result.valid:
            self.audit_logger.log_security_event(
                event='kb_load_failed',
                reason='signature_verification_failed',
                kb_path=kb_path
            )
            raise SecurityException("知识库签名验证失败")
        
        # 4. 检查一致性
        consistency_issues = self.integrity_validator.check_consistency(knowledge_base)
        
        if any(issue.type == 'missing_dependency' for issue in consistency_issues):
            self.audit_logger.log_security_event(
                event='kb_consistency_warning',
                issues=[issue.message for issue in consistency_issues]
            )
        
        # 5. 记录审计日志
        self.audit_logger.log_access(
            action='load_knowledge_base',
            kb_path=kb_path,
            hash=integrity_result.hash
        )
        
        return SecureLoadResult(
            knowledge_base=knowledge_base,
            integrity=integrity_result,
            consistency_issues=consistency_issues
        )
```

```python
class OutputSecurityValidator:
    """输出安全验证器"""
    
    def validate_generated_code(self, code: str, constraints: Dict) -> ValidationResult:
        """验证生成的代码安全性"""
        
        result = ValidationResult(valid=True, violations=[])
        
        # 检查must_not约束是否被违反
        for constraint in constraints.get('must_not', []):
            if constraint.get('type') == 'security':
                pattern = constraint.get('pattern', '')
                if pattern and re.search(pattern, code):
                    result.valid = False
                    result.violations.append({

                        'reason': '安全约束被违反'

        # 额外安全扫描
        security_issues = self._scan_security_issues(code)
        result.violations.extend(security_issues)
        
        return result
    
    def _scan_security_issues(self, code: str) -> List[Dict]:
        """扫描代码中的安全问题"""
        
        issues = []
        
        # XSS检查
        if re.search(r'innerHTML\s*=', code) and not re.search(r'DOMPurify', code):
            issues.append({
                'type': 'potential_xss',
                'description': '使用innerHTML但未发现净化处理'
            })
        
        # 敏感数据暴露检查
        if re.search(r'(password|token|api_key)\s*[=:]\s*["\'][^"\']+["\']', code):
            issues.append({
                'type': 'sensitive_data_exposure',
                'description': '代码中可能包含硬编码的敏感数据'
            })
        
        return issues
```

```python
class MemoryLeakDetector:
    """长时间运行内存泄漏检测器"""
    
    def run_extended_test(
        self, 
        duration_hours: int = 24,
        task_interval_seconds: int = 30
    ) -> MemoryLeakReport:
        """执行长时间内存泄漏测试"""
        
        import tracemalloc
        tracemalloc.start()
        
        memory_samples = []
        baseline = None
        task_count = 0
        
        start_time = time.time()
        end_time = start_time + duration_hours * 3600
        
        while time.time() < end_time:
            # 执行一个完整任务周期
            self._execute_task_cycle()
            task_count += 1
            
            # 采样内存使用
            current, peak = tracemalloc.get_traced_memory()
            
            if baseline is None:
                baseline = current
            
            memory_samples.append({
                'time': time.time() - start_time,
                'current_mb': current / 1024 / 1024,
                'peak_mb': peak / 1024 / 1024,
                'delta_mb': (current - baseline) / 1024 / 1024
            })
            
            # 每100个任务强制GC
            if task_count % 100 == 0:
                gc.collect()
            
            time.sleep(task_interval_seconds)
        
        tracemalloc.stop()
        
        return self._analyze_memory_trend(memory_samples, task_count)
    
    def _analyze_memory_trend(
        self, 
        samples: List[Dict], 
        task_count: int
    ) -> MemoryLeakReport:
        """分析内存趋势，判断是否存在泄漏"""
        
        # 线性回归分析
        times = [s['time'] for s in samples]
        memories = [s['current_mb'] for s in samples]
        
        slope, intercept = np.polyfit(times, memories, 1)
        
        # 判断泄漏标准：斜率 > 0.01 MB/小时
        leak_threshold = 0.01 / 3600  # MB/秒
        has_leak = slope > leak_threshold
        
        return MemoryLeakReport(
            has_memory_leak=has_leak,
            leak_rate_mb_per_hour=slope * 3600,
            baseline_mb=samples[0]['current_mb'],
            final_mb=samples[-1]['current_mb'],
            peak_mb=max(s['peak_mb'] for s in samples),
            total_tasks=task_count,
            duration_hours=samples[-1]['time'] / 3600,
            recommendation=self._generate_recommendation(slope) if has_leak else None
        )
```

```python
class ConstraintPerformanceOptimizer:
    """约束性能优化器"""
    
    def __init__(self):
        self.constraint_cache = LRUCache(maxsize=1000)
        self.knowledge_base_index = {}
    
    def optimize_constraint_generation(
        self,
        requirement: Dict,
        context: Dict
    ) -> Constraints:
        """优化约束生成"""
        
        # 1. 检查缓存
        cache_key = self._generate_cache_key(requirement, context)
        if cache_key in self.constraint_cache:
            return self.constraint_cache[cache_key]
        
        # 2. 增量生成
        base_constraints = self._get_base_constraints(requirement['page_type'])
        delta_constraints = self._generate_delta(requirement, base_constraints)
        
        # 3. 合并并缓存
        result = self._merge_constraints(base_constraints, delta_constraints)
        self.constraint_cache[cache_key] = result
        
        return result
    
    def parallel_validate(
        self,
        code: str,
        constraints: Constraints
    ) -> ValidationResult:
        """并行验证"""
        
        with ThreadPoolExecutor(max_workers=4) as executor:
            # 并行执行must_follow和must_not验证
            mf_future = executor.submit(
                self._validate_must_follow, 
                code, 
                constraints['must_follow']
            )
            mn_future = executor.submit(
                self._validate_must_not,
                code,
                constraints['must_not']
            )
            
            mf_result = mf_future.result()
            mn_result = mn_future.result()
```

```python
class VersionMigrationHandler:
    """版本迁移处理器"""
    
    def check_migration_needed(
        self, 
        current_version: str, 
        target_version: str
    ) -> MigrationPlan:
        """检查是否需要迁移并生成迁移计划"""
        
        current = parse_version(current_version)
        target = parse_version(target_version)
        
        if target.major > current.major:
            return MigrationPlan(
                mode='cold_migration',
                breaking_changes=True,
                estimated_downtime='5-10分钟'
            )
        elif target.minor > current.minor:
            return MigrationPlan(
                mode='hot_migration',
                breaking_changes=False,
                estimated_downtime='0秒'
            )
        else:
            return MigrationPlan(
                mode='hot_migration',
                breaking_changes=False,
                estimated_downtime='0秒'
            )
    
    def execute_migration(
        self, 
        plan: MigrationPlan,
        knowledge_base_path: str
    ) -> MigrationResult:
        """执行迁移"""
        
        if plan.mode == 'cold_migration':
            self._pause_task_reception()
            self._wait_pending_tasks(timeout=300)
        
        # 加载新版本知识库
        new_kb = self._load_knowledge_base(knowledge_base_path)
        
        # 验证约束兼容性
        compatibility_report = self._validate_compatibility(new_kb)
        
        if not compatibility_report.success:
            return MigrationResult(
                success=False,
                reason=compatibility_report.errors
            )
        
        # 更新缓存
        self._update_cache(new_kb)
        
        if plan.mode == 'cold_migration':
            self._resume_task_reception()
        
        return MigrationResult(success=True)
```

```python
class ErrorRecoveryManager:
    """错误恢复管理器"""
    
    RECOVERY_STRATEGIES = {
        'constraint_parse_error': {
            'strategy': 'fallback_to_default',
            'fallback_constraint': {
                'type': 'generic',
                'description': '使用通用约束',
                'severity': 'warning'
            }
        },
        'constraint_not_found': {
            'strategy': 'generate_dynamic',
            'generator': 'generate_constraint_from_context'
        },
        'validation_timeout': {
            'strategy': 'partial_validation',
            'max_time': 30,
            'priority': 'error_only'
        },
        'knowledge_base_unavailable': {
            'strategy': 'use_cache',
            'cache_ttl': 3600,
            'fallback': 'minimal_constraints'
        }
    }
    
    def handle_error(
        self, 
        error: ConstraintError, 
        context: TaskContext
    ) -> RecoveryResult:
        """处理约束错误并尝试恢复"""
        
        error_type = error.__class__.__name__
        strategy_config = self.RECOVERY_STRATEGIES.get(error_type, {})
        
        if not strategy_config:
            return RecoveryResult(
                success=False,
                reason=f"无恢复策略: {error_type}"
            )
        
        strategy = strategy_config['strategy']
        
        if strategy == 'fallback_to_default':
            return self._apply_fallback(context, strategy_config)
        elif strategy == 'generate_dynamic':
            return self._generate_dynamic(context, strategy_config)
        elif strategy == 'partial_validation':
            return self._partial_validation(context, strategy_config)
        elif strategy == 'use_cache':
            return self._use_cache(context, strategy_config)
        
        return RecoveryResult(success=False, reason="未知恢复策略")
    
    def _apply_fallback(
        self, 
        context: TaskContext, 
        config: Dict
    ) -> RecoveryResult:
        """应用回退约束"""
        
        fallback = config['fallback_constraint']
        
        logging.warning(
            f"约束解析失败，使用回退约束: {fallback['description']}",
            extra={'task_id': context.task_id}
        )
        
        return RecoveryResult(
            success=True,
            constraint=fallback,
            warning="使用了回退约束，可能影响设计一致性"
        )
```

## 结论

### 主要贡献总结

本研究围绕"多Agent系统中设计一致性保障"这一核心问题，提出了一套完整的设计系统驱动的约束传播机制。主要贡献包括：

**理论贡献**：

1. **约束传播链理论**：建立了Design-System → AES → Agent → Code的四层约束传播模型，明确了各层的职责和语义保持规则。

2. **execution_constraints语义定义**：精确定义了must_follow和must_not约束的语义、类型和验证方法，为AES作为约束载体提供了理论基础。

3. **设计系统一致性测量理论**：将设计系统合规性纳入PDA测量体系，定义了B5指标的测量公式和阈值标准。

**工程贡献**：

1. **Agent-friendly知识库格式**：设计了基于JSON Schema扩展的设计系统知识库格式，优化了Agent查询效率。

2. **约束生成算法**：实现了must_follow和must_not约束的自动生成算法，支持冲突检测和消解。

3. **架构集成设计**：将设计系统驱动能力融入AES任务载体、Task/Subagent架构、硬性卡点机制和Eval-Driven控制平面。

4. **工程实践验证**：通过电商登录页面和数据仪表盘两个案例，验证了方法的有效性，设计一致性从62%提升至95%。

### 未来研究方向

1. **多模态输入扩展**：研究从UI设计稿图像直接生成约束的技术，提升用户体验。

2. **动态约束调整**：研究运行时根据用户反馈动态调整约束的机制。

3. **跨组织互操作**：研究不同设计系统之间的Schema映射和令牌对齐方法。

4. **AI辅助设计决策**：研究AI在约束生成过程中的推理和决策能力。

## 附录C：约束描述语言（CDL）规范

### C.1 CDL概述

约束描述语言（Constraint Description Language, CDL）是一种形式化的约束表达语言，用于精确定义设计系统约束，提高Agent的可解释性和验证准确性。

### C.2 语法定义

```ebnf
(* CDL语法 - EBNF定义 *)

constraint_doc    ::= { statement } ;

statement         ::= must_follow_stmt | must_not_stmt | import_stmt ;

must_follow_stmt  ::= "MUST_FOLLOW" constraint_id "{" constraint_body "}" ;
must_not_stmt     ::= "MUST_NOT" constraint_id "{" constraint_body "}" ;

constraint_id     ::= IDENTIFIER ;

constraint_body   ::= 
    "type" ":" constraint_type ","
    "description" ":" STRING ","
    "validation" ":" validation_expr ","
    [ "severity" ":" severity_level "," ]
    [ "scope" ":" scope_expr "," ]
    [ "meta" ":" meta_obj ] ;

constraint_type   ::= "component" | "token" | "accessibility" | "pattern" | "security" ;

validation_expr   ::= 
    pattern_match | 
    presence_check | 
    range_check | 
    compound_expr ;

pattern_match     ::= "MATCH" STRING [ "IN" scope_expr ] ;
presence_check    ::= "PRESENT" element_ref ;
range_check       ::= "RANGE" "(" expr "," expr ")" ;
compound_expr     ::= "AND" "(" validation_expr { "," validation_expr } ")" |
                      "OR" "(" validation_expr { "," validation_expr } ")" |
                      "NOT" "(" validation_expr ")" ;

severity_level    ::= "error" | "warning" | "info" ;

element_ref       ::= "ELEMENT" IDENTIFIER | "CSS" property_name | "ATTR" attr_name ;

scope_expr        ::= "ALL" | "HTML" | "CSS" | "JS" | component_ref ;
component_ref     ::= "COMPONENT" IDENTIFIER ;
```

### C.3 CDL示例

```cdl
// 设计系统约束定义示例
```

### C.4 CDL解析器设计

```python
class CDLParser:
    """约束描述语言解析器"""
    
    def __init__(self):
        self.lexer = CDLLexer()
        self.ast_builder = ASTBuilder()
    
    def parse(self, cdl_source: str) -> List[Constraint]:
        """解析CDL源码，返回约束列表"""
        
        # 词法分析
        tokens = self.lexer.tokenize(cdl_source)
        
        # 语法分析，构建AST
        ast = self.ast_builder.build(tokens)
        
        # 语义分析，转换为约束对象
        constraints = self._analyze(ast)
        
        return constraints
    
    def _analyze(self, ast: CDLAST) -> List[Constraint]:
        """语义分析，将AST转换为约束对象"""
        
        constraints = []
        
        for stmt in ast.statements:
            if stmt.type == 'must_follow':
                constraint = self._build_must_follow(stmt)
                constraints.append(constraint)
            elif stmt.type == 'must_not':
                constraint = self._build_must_not(stmt)
                constraints.append(constraint)
        
        return constraints
    
    def _build_must_follow(self, stmt: Statement) -> Constraint:
        """构建must_follow约束"""
        
        return Constraint(
            id=stmt.constraint_id,
            type=stmt.body['type'],
            description=stmt.body['description'],
            validation=self._compile_validation(stmt.body['validation']),
            severity=stmt.body.get('severity', 'error'),
            scope=stmt.body.get('scope', 'ALL'),
            meta=stmt.body.get('meta', {})
        )
    
    def _compile_validation(self, validation_expr: str) -> ValidationPattern:
        """编译验证表达式为可执行的正则模式"""
        
        # 转换CDL验证表达式为Python正则
        # 例如: MATCH "<Button[^>]*>" IN HTML
        # 转换为: re.search(r'<Button[^>]*>', html_content)
        
        return ValidationPattern(
            pattern=self._extract_pattern(validation_expr),
            scope=self._extract_scope(validation_expr),
            negated='NOT' in validation_expr
        )


class CDLValidator:
    """CDL验证器"""
    
    def validate_code(
        self, 
        code: Dict[str, str], 
        constraints: List[Constraint]
    ) -> ValidationResult:
        """验证代码是否符合CDL定义的约束"""
        
        result = ValidationResult(passed=True, violations=[], warnings=[])
        
        for constraint in constraints:
            check_result = self._check_constraint(code, constraint)
            
            if not check_result.passed:
                if constraint.severity == 'error':
                    result.violations.append(check_result)
                    result.passed = False
                else:
                    result.warnings.append(check_result)
        
        return result
    
    def _check_constraint(
        self, 
        code: Dict[str, str], 
        constraint: Constraint
    ) -> CheckResult:
        """检查单个约束"""
        
        validation = constraint.validation
        scope_content = self._get_scope_content(code, validation.scope)
        
        if validation.negated:
            # MUST_NOT: 模式不应匹配
            matched = re.search(validation.pattern, scope_content)
            return CheckResult(
                passed=not bool(matched),
                constraint_id=constraint.id,
                description=constraint.description,
                actual=f"找到匹配: {matched.group()}" if matched else "无匹配"
            )
        else:
            # MUST_FOLLOW: 模式应匹配
            matched = re.search(validation.pattern, scope_content)
            return CheckResult(
                passed=bool(matched),
                constraint_id=constraint.id,
                description=constraint.description,
                actual=f"找到匹配: {matched.group()}" if matched else "未找到匹配"
            )
```
