### 执行摘要

本规划基于对**15+份核心文献**的深度调研，包括：
- OpenClaw 源码架构分析（MAPE-K 实现、记忆系统、Lane Queue、7 层安全）
- 竞品 PDA 指标对标（Claude Code 实证数据、行业共识框架）
- Safety-II 企业案例（NASA/NHS/OpenAI Harness Engineering 转型路径）

**核心判断**：
- OpenCode 当前处于**HIL L2-L3**（监督 - 协作级），架构处于**Skill→Agent 相变边缘**
- 与 OpenClaw 相比存在**8 项关键差距**（记忆系统、多代理协作、协议元数据、PDA 指标等）
- 追赶 OpenClaw**符合最终目标**，但必须以 DACES 理论为指导，同步建设测量/协议/风险运营层

**进化策略**：
- **方案 C**：MAPE-K 闭环 + PDA 指标双轨并行
- **Safety-II 范式**：从防失败转向保成功
- **双协议架构**：MCP（工具集成）+ A2A（多 Agent 协作）

| **风险控制** | 审计缺失率 | 缺失日志数/总操作数×100% | 0% | 0% |
| **组织协同** | 责任归因完成率 | 可归因决策数/总决策数×100% | 100% | 100% |
| **组织协同** | 人工兜底成功率 | 成功数/接管总数×100% | >95% | >95% |

**PDA 综合评分公式**：
```
PDA_Score = 0.35×Task_Autonomy + 0.25×Trajectory_Quality 
          + 0.25×Risk_Control + 0.15×Human_Oversight

通过阈值：PDA_Score ≥ 75 且 Risk_Control ≥ 90
```

## 理论支柱

### 1. MAPE-K 框架（IBM 2003, 8000+ 引用）

```
Monitor（监控）→ Analyze（分析）→ Plan（规划）→ Execute（执行）+ Knowledge（知识）

四大自*特性:
- 自配置（Self-Configuring）
- 自愈合（Self-Healing）
- 自优化（Self-Optimizing）
- 自保护（Self-Protecting）
- 自演进（Self-Evolving）← 2025 年新增
```

### 2. STPA 安全分析（MIT Leveson, 8000+ 引用）

```
传统安全：组件故障 → 事件链 → 事故
STPA 安全：控制回路不安全交互 → 安全约束违反 → 事故

四类不安全控制动作 (UCA):
1. 提供控制动作但未提供（遗漏）
2. 未提供控制动作但应提供（缺失）
3. 提供错误/不当的控制动作
4. 控制动作时机/顺序/持续时间错误
```

### 3. 韧性工程（Safety-I → Safety-II）

| 维度 | Safety-I（当前） | Safety-II（目标） |
|------|-----------------|------------------|
| 安全定义 | 无事故、无故障 | 功能持续成功 |
| 关注焦点 | 失败、错误、故障 | 成功、适应、调整 |
| 人的角色 | 风险源、需要控制 | 资源、需要赋能 |

四大核心能力：响应、监控、学习、预期

### 4. PDA 指标（SAE J3016 + NIST TEVV + Anthropic 实证）

自主性等级测量 + 风险联合评估矩阵

### 5. MCP/A2A 协议（Anthropic + Google/Linux 基金会）

- MCP：7158+ 服务器，60%+ 大型企业采用
- A2A：150+ 企业支持，50 万+ 兼容 Agent
