# 从Prompt Engineering到Harness Engineering：AI Agent架构演进综述

## 摘要

本文系统梳理了AI Agent技术架构从2022年至今的演进路径，涵盖Prompt Engineering、Skill-based Agent架构、Harness Engineering三个核心阶段。通过对比各阶段的技术特征、核心问题与解决方案，揭示AI协作范式从"微观管理"到"制度化授权"的转变趋势，为未来AI系统架构设计提供理论参考。

---

## 二、第一阶段：Prompt Engineering（2022-2023）

### 2.1 核心特征
| 维度 | 描述 |
|------|------|
| **关注点** | 提示词模板与技巧优化 |
| **交互模式** | 单轮任务，人类主导 |
| **假设前提** | 模型即产品，写好prompt就能输出正确结果 |
| **典型产出** | "万能模板"、"高阶技巧"库 |

### 2.2 技术局限
- **上下文碎片化**：每次对话独立，缺乏状态保持
- **能力边界模糊**：无法明确界定模型能/不能做什么
- **不可复现**：相同prompt在不同场景下输出差异大
- **人类认知负荷高**：需要持续微调prompt以获得稳定输出

### 2.3 代表性实践
- Chain-of-Thought提示
- Few-shot Learning模板
- Role-playing提示框架

## 三、第二阶段：Context Engineering（2023-2024）

### 3.1 核心突破
意识到模型输出受**整个信息环境**影响，包括：
- 系统规则与约束
- 领域文档与知识库
- 工具集与API接口
- 历史对话与记忆

### 3.2 技术架构
```
┌─────────────────────────────────────┐
│           Context Layer             │
│  ┌─────────┬─────────┬─────────┐   │
│  │  Rules  │  Docs   │  Tools  │   │
│  └─────────┴─────────┴─────────┘   │
│              ↓                      │
│         LLM + Memory               │
└─────────────────────────────────────┘
```

### 3.3 关键进展
- **RAG技术成熟**：检索增强生成成为标准配置
- **向量数据库普及**：Pinecone、Weaviate等成为基础设施
- **记忆机制引入**：短期/长期记忆分离设计

## 四、第三阶段：Skill-based Agent架构（2024-2025）

### 4.1 核心问题：Agent的"高智商困境"

Anthropic工程师Barry Zhang的经典比喻：
> "你希望谁帮你报税？是智商300的数学天才Mahesh，还是经验丰富的税务专家Barry？"

**悖论**：Agent能写诗、能编程、能推理，但在特定领域任务上缺乏**可持续、可复用、可验证的程序性专业知识**。

### 4.2 Skill架构定义

| 层级 | 类比 | 职责 |
|------|------|------|
| Model | CPU | 提供原始推理能力（如Claude 3.5） |
| Agent Runtime | OS | 管理文件系统、代码执行、上下文调度 |
| MCP Servers | 外设驱动 | 连接外部系统（数据库、API、搜索引擎） |
| **Skills Library** | **应用软件** | **编码领域专用能力包** |

### 4.3 Skill系统核心特性

```
┌──────────────────────────────────────────────┐
│              Skills Directory                │
│         （AI领域的App Store）                │
│  ┌─────────┬─────────┬─────────┬─────────┐  │
│  │ 发现    │ 分发    │ 评价    │ 版本    │  │
│  └─────────┴─────────┴─────────┴─────────┘  │
└──────────────────────────────────────────────┘
```

**关键能力**：
1. **可发现性**：统一注册与检索机制
2. **可组合性**：Skill之间可串联/并联调用
3. **可验证性**：每个Skill有明确的输入/输出规范与测试用例
4. **可继承性**：经验可传递给"未来的自己"

### 4.4 行业进展
- **2025年12月**：Anthropic发布Agent Skills开放标准，SDK上线agentskills.io
- **微软**：VS Code与GitHub集成Skills标准
- **OpenAI**：Codex文档增补Skills支持
- **生态工具**：Cursor、Goose、OpenCode等主流编码Agent跟进适配

## 五、第四阶段：Harness Engineering（2025-2026）

### 5.1 概念定义

**Harness Engineering（智能体外壳工程）**：设计一个环境，让AI能可靠地完成复杂工作。

> 2026年2月5日，HashiCorp联合创始人Mitchell Hashimoto正式命名此实践。六天后，OpenAI发布内部实验报告直接使用此术语。

### 5.2 核心认知转变

| 旧认知 | 新认知 |
|--------|--------|
| 模型是产品本身 | 模型只是大脑 |
| 关注prompt优化 | 关注运行环境设计 |
| 人类持续干预 | 制度化授权与约束 |

### 5.3 Harness核心维度

```
┌─────────────────────────────────────────────────┐
│              Harness Engineering                │
│  ┌───────────┬───────────┬───────────┐         │
│  │  约束层   │  反馈层   │  安全层   │         │
│  │  - 模块隔离│  - 测试验证│  - 权限控制│         │
│  │  - 变更限制│  - 自动回滚│  - 操作审计│         │
│  └───────────┴───────────┴───────────┘         │
│                    ↓                            │
│            Agent + Skills                       │
└─────────────────────────────────────────────────┘
```

### 5.4 典型实践案例

**OpenAI实验数据**：
- 5个月内用Codex写100万行生产级代码
- 3个工程师平均每天合并3.5个PR
- **没有一行代码是工程师手写的**

**关键Harness设计**：
1. 先给Agent注入架构文档
2. 限制每次只能改一个模块
3. 强制改完后必须跑测试
4. 检测到失败时自动回滚
5. Agent花了2天，成功重构了80%的代码

**LangChain案例**：
- Terminal Bench 2.0上从52.8%跃升至66.5%
- **只改了harness，模型没动**

### 5.5 常见误区

> ❌ 把harness当作"给LLM接工具的胶水层"
> 
> ✅ Harness是**生产环境运行系统**，决定模型能否稳定执行、长会话运行、避免危险操作

## 六、演进逻辑与管理学映射

### 6.1 人与AI协作范式转变

引入管理学**情境领导理论**，揭示人与AI关系的演进：

| 阶段 | 人类角色 | AI角色 | 管理风格 |
|------|----------|--------|----------|
| Prompt Engineering | 微观管理者 | 执行者 | 指令式 |
| Context Engineering | 知识管理者 | 助手 | 辅导式 |
| Skill-based Agent | 能力设计者 | 专业员工 | 支持式 |
| Harness Engineering | 意图设定者 | 自主代理 | 授权式 |

### 6.2 核心洞察

> **Prompt Engineering管的是「说什么」，Context Engineering管的是「知道什么」，Skill-based Agent管的是「能做什么」，Harness Engineering管的是「在什么环境里做事」。**

## 七、技术挑战与未来方向

### 7.1 当前挑战

| 挑战领域 | 具体问题 |
|----------|----------|
| **可靠性** | 长任务执行中的错误累积与传播 |
| **安全性** | 自主操作的风险边界界定 |
| **可解释性** | 复杂决策链的追溯与审计 |
| **标准化** | Skill与Harness的跨平台互操作 |

### 7.2 未来研究方向

1. **自适应Harness**：根据任务复杂度动态调整约束强度
2. **Skill市场生态**：建立可发现、可分发、可评价的技能经济
3. **多Agent协同Harness**：跨Agent的协调与冲突解决机制
4. **人机共治框架**：人类干预的触发条件与介入方式

---

## 八、结论

从Prompt Engineering到Harness Engineering的演进，本质上是**AI工程化成熟度**的提升过程：

1. **从关注模型到关注系统**：认识到模型能力只是系统能力的一部分
2. **从人工干预到制度约束**：用环境设计替代持续的人工监督
3. **从单点优化到架构演进**：建立可扩展、可演化的技术框架

**核心启示**：决定AI Agent上限的，不再是模型本身，而是**环境设计能力**。这正成为工程师的新护城河。
