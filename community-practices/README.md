# 社区实践案例 (Community Practices)

欢迎提交使用本理论框架的实践案例！

---

## 提交方式 (How to Contribute)

1. **Fork 本仓库** - 点击右上角 "Fork" 按钮
2. **创建你的案例目录** - 在 `community-practices/` 下新建文件夹，命名格式：`<你的用户名>-<项目简述>`
3. **添加案例文档** - 至少包含 `README.md`，描述你的实践过程和结果
4. **提交 Pull Request** - 点击 "Contribute" → "Open Pull Request"

---

## 案例模板 (Case Template)

你的 `README.md` 应包含：

```markdown
# 案例名称

## 应用场景
描述你的项目/系统背景

## 应用的框架
- [ ] DACES 架构模式
- [ ] HWAT 工作流
- [ ] AEAS 评估体系
- [ ] 其他：_____

## 实践过程
描述如何应用本理论框架

## 结果与验证
实践成果、数据、或观察

## 待改进点
遇到的问题或需要完善的地方
```

---

## 审核说明 (Review Process)

- 所有提交需经过仓库维护者审核
- 审核通过后才会合并到主仓库
- 如不符合要求，可能会被关闭或要求修改

提交的案例将遵循本仓库的默认许可条款（All Rights Reserved），原作者保留署名权。

---

## 常见问题

**Q: 我的案例必须完全遵循框架吗？**  
A: 不需要，部分应用也可以。但请说明哪些部分被应用/修改了。

**Q: 我可以提交商业项目案例吗？**  
A: 可以，但请确保不泄露敏感信息。

**Q: 别人可以修改我的案例吗？**  
A: 其他人可以通过 PR 提出修改建议，但需经过你和维护者同意。
